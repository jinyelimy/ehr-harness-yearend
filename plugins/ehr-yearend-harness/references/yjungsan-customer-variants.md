# yjungsan 고객/회사별 분기 사전

연말정산 도메인에서 고객사·회사별·그룹별로 적용되는 분기 로직(선배포·커스텀·예외 처리)이 있을 *가능성*을 탐색하기 위한 reference. 분기 사실 자체가 아니라 **어디를 먼저 봐야 하는지** 알려주는 사전.

> 본 사전은 `yearend-chain-tracer` 의 **고객/회사별 분기 보조 확인** 단계와 `yearend-investigator` 의 **자유텍스트 후속조치 입력 처리** 단계에서 사용된다. 비어 있는 슬롯이 정상 — 첫 사용 시 사용자/AI 가 점진적으로 채워간다.

---

## 사용 방법

1. 사용자 입력에서 고객/회사 키워드 추출 (예: "GS리테일 영향 봐줘", "카카오뱅크 분기 있나", "선배포 케이스")
2. 본 사전의 슬롯에서 매칭 확인 (실명 또는 익명 식별자)
3. 매칭 있으면: 탐색 키워드 + 확인 위치 안내 (단정 X, "확인 필요" 표시)
4. 매칭 없으면: 추측하지 않고 "매칭 없음 — 추측 안 함" 으로 표시

---

## 원칙

- 본 문서는 고객/회사별 분기 *가능성*을 탐색하기 위한 reference 다. 분기 사실 자체가 아니라 *어디를 봐야 하는지* 만 기록한다.
- 공개 저장소에 올릴 수 없는 정보는 기록하지 않는다.
- 장애 상세, 운영 데이터, 계약 조건, 담당자명, 내부 배포 일정은 기록하지 않는다.
- **기본 슬롯은 익명 고객명으로 둔다** (`고객A`, `고객B`, …).
- 실명 고객명은 사용자 또는 조직이 *공개 가능*하다고 명시한 경우에만 기록한다.
- 확인 가능한 코드 경로, 파일명, 패키지명, 매퍼명, 검색 키워드 중심으로 기록한다.
- 매칭이 없으면 고객별 분기를 추측하지 않는다. "고객X는 별도 로직 적용됨" 같은 단정 금지.
- 익명화는 이름 직접 노출은 막지만 업계 추측까지 차단하지는 않는다. 분기 *내용*(탐색 키워드·확인 위치)도 일반 코드 식별자 수준으로 유지해 그 자체로는 식별 가치가 없도록 한다.

---

## 슬롯 형식

| 고객/그룹 | 탐색 키워드 | 분기 유형 | 확인 위치 | 검증 관점 | 마지막 확인 |
|---|---|---|---|---|---|
| 고객A (유통계열) | _(미채움)_ | 화면 / 매퍼 / 패키지 / report / 패치 | _(미채움)_ | _(미채움)_ | _(미채움)_ |
| 고객B (금융계열) | _(미채움)_ |  |  |  |  |
| 고객C (항공계열) | _(미채움)_ |  |  |  |  |
| 고객D (제조계열) | _(미채움)_ |  |  |  |  |
| 고객E (플랫폼·기타) | _(미채움)_ |  |  |  |  |

각 컬럼:

- **고객/그룹**: 익명 식별자(default) 또는 사용자 명시 시 실명. 계열명만으로도 일정 추측이 가능하므로 *내용*으로 차폐를 강화한다.
- **탐색 키워드**: grep 할 때 쓸 코드 식별자 수준 (예: `BefComMgr.jsp 의 EXCEPT_BP_CD 분기`). 회사명 직접 노출은 피한다.
- **분기 유형**: 화면 / 매퍼 / 패키지 / report / 패치 / 외부 인터페이스 등 카테고리.
- **확인 위치**: 파일 경로, 패키지 이름, 매퍼 ID 등 코드에서 검증 가능한 위치만.
- **검증 관점**: 어떤 시나리오를 돌리면 분기를 확인할 수 있는지 (한 줄).
- **마지막 확인**: `YYYY-MM-DD (담당자ID)` — 분기가 살아 있는지 마지막 확인 시점.

---

## 슬롯 채우는 방법

행을 채울 때 다음 형식을 따른다.

```
| 고객A (유통계열) | EXCEPT_BP_CD, BefComMgr 분기 | 화면 / 매퍼 | src/main/java/.../BefComMgrController.java:120 + Mapper.xml 의 specificQuery | 종전근무지 합산 화면에서 사업장 코드 분기 확인 | 2026-04-30 (jinyelimy) |
```

- 분기 *내용*은 코드 식별자 수준 (사업장 코드 값, 외부 API URL 같은 운영 데이터는 기록 X).
- 실명을 박는 경우 사용자 명시 승인 필요.
- 추가 전, 코드/매퍼/패키지에서 **실제로 분기가 존재함을 확인**한 경우에만 기록. "들어보니 그렇다더라" 정보는 기록하지 않는다.

---

## 누락된 슬롯 발견 시

AI 가 키워드 매칭에 실패해 사용자에게 "본 적 있나요?" 역질문을 했고 사용자가 위치를 알려주면, 다음 한 줄을 본 사전에 PR 로 추가하자고 사용자에게 제안한다.

```
| <고객 식별자(익명 default)> | <탐색 키워드 콤마 구분> | <분기 유형> | <확인 위치> | <검증 관점 한 줄> | YYYY-MM-DD (사용자 ID) |
```

> 단정 금지: 추가하기 전, 코드/매퍼/패키지에서 **실제로 분기가 존재함을 확인**한 경우에만 기록한다.
