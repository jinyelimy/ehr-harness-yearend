# yjungsan 테스트 데이터 위치 사전

연말정산 도메인의 영향 분석·쿼리 작성 결과를 검증할 때 사용할 **테스트 데이터의 위치**를 정리한다.
실제 데이터 *내용*이 아니라 *어디에 있는지* + *어떻게 사용하는지* 의 인덱스다.

> 본 사전은 `yearend-chain-tracer` 의 **테스트 데이터 위치 안내** 단계에서 사용된다.
> 누락된 시나리오가 발견되면 점진적으로 추가한다 (PR 환영).

---

## 사용 방법

1. 사용자 시나리오에서 키워드 추출 (예: "출산지원금 + 외국인 + 2026 귀속")
2. 본 사전의 **시나리오 슬롯**에서 매칭 확인
3. 매칭 있으면: 위치 + 사용 방법 안내
4. 매칭 없으면: **탐색 경로** 의 표준 위치를 grep 후 후보 보고 + 사용자에게 "본 적 있는지" 역질문

## 탐색 경로 (타깃 EHR 프로젝트 기준)

> 본 플러그인은 *타깃 EHR 프로젝트* 위에 설치되므로 실제 위치는 프로젝트마다 다를 수 있다.
> 첫 사용 시 다음 위치를 순서대로 grep 해 발견된 위치를 본 사전에 추가한다.

| 우선순위 | 위치 패턴 | 일반적 내용 | 비고 |
|---|---|---|---|
| 1 | `src/test/resources/**/*.sql` | 단위/통합 테스트용 fixture SQL | Spring/JUnit 표준 위치 |
| 2 | `src/test/resources/**/*.xml` | DBUnit dataset, 테스트용 매퍼 결과 stub | DBUnit 사용 프로젝트 |
| 3 | `**/test-data/**/*.sql`, `**/testdata/**/*.sql` | 테스트 전용 더미 INSERT 스크립트 | 사내 컨벤션 |
| 4 | `docs/**/sample*.sql`, `docs/**/시나리오*.sql` | 시나리오별 샘플 SQL | 분석 문서 옆 |
| 5 | `EHR_HR50/sql/yjungsan/**/*.sql` 또는 사내 SQL 저장소 | 운영 더미 데이터 / 시나리오별 INSERT 묶음 | 가장 흔한 실무 위치 |
| 6 | DB 스키마 자체의 테스트 사원 (`SABUN` 이 `T0001`, `TEST00` 등으로 시작) | 운영 DB 안에 직접 보관된 테스트 사원 | 운영/테스트 분리 안 된 환경에서 흔함 |

> 실제 발견된 위치는 아래 **시나리오별 슬롯** 표에 누적 기록한다.

## 시나리오별 검증 체크리스트 (템플릿)

영향 분석·코드 변경 후 *어떤 항목을 확인하면 검증이 끝나는가*. 후속조치 처리 시 누락 방지용 표준 체크리스트.

> 본 체크리스트는 사용자가 직접 점검하는 가이드다. AI 는 항목별 SELECT/DESC 쿼리 작성을 도와줄 수 있으나, INSERT/UPDATE/DELETE 는 db-read-only 훅이 차단한다.

### 퇴직소득 변경

- [ ] 대상자 식별 — 귀속연도·사번·퇴직일·정산구분(`ADJUST_TYPE`) 확인
- [ ] 입력 원천자료 반영 (`TCPN8##` 원천 레이어)
- [ ] 퇴직소득 세액 계산 결과 (`TCPN843` 등 계산결과 레이어)
- [ ] 퇴직소득원천징수영수증 출력 (PDF 반영 체인)
- [ ] 지급조서 / 원천세 신고 영향
- [ ] 마감 전/후 재계산 (`ADJUST_TYPE='88'`) 영향
- [ ] EHR4 / EHR5 경로 차이
- [ ] 가산세 계산 영향 (`yjungsan-tax-calc-rules.md` §2)

### 출산지원금 반영

- [ ] 원천자료 반영 — `TCPN887` 또는 관련 원천 테이블에 입력 확인
- [ ] 비과세 / 과세 구분 정확성 — 과세급여 이중 차감 여부 점검 (`yjungsan-tax-calc-rules.md` §1)
- [ ] 계산 결과 테이블 반영 (`PKG_CPN_YEA_{YY}_CALC` 호출 후 `TCPN843` 정합)
- [ ] 원천징수영수증 출력 항목 표시
- [ ] PDF 반영 체인 (`P_CPN_YEA_PDF_*`) 영향
- [ ] 마감 후 재계산 시 결과 일관성

### 종전근무지 합산

- [ ] 원천자료 — `TCPN817` / `TCPN818` 입력 확인
- [ ] 화면 — `BefComMgr` 입력 / 조회 정합성
- [ ] 합산 후 총급여 · 과세급여 · 산출세액 정합
- [ ] 종전근무지 PDF 반영 (`TCPN851` / `TCPN855`)
- [ ] 오류검증 — `TCPN849` 신규 오류 미발생
- [ ] 마감 영향 — 합산 변경 후 재계산 흐름

### 외국인 분납

- [ ] 대상자 식별 — `FOREIGNER_YN='Y'` + 분납 대상 조건
- [ ] 분납 회차별 결제 금액 계산
- [ ] `TCPN8##` 분납 관련 컬럼 반영 (해당 컬럼은 매퍼 · `DESC` 로 컬럼 검증 게이트 통과 필수)
- [ ] 원천징수영수증 분납 항목 표시
- [ ] 마감 / 가산세 영향 — 분납 미납 시 가산세 계산 (`yjungsan-tax-calc-rules.md` §2)
- [ ] 외국인 단일세율 vs 일반세율 분기 정합성

> 누락된 시나리오 템플릿은 **누락된 슬롯 발견 시** 절차로 추가한다. 단, 체크리스트 항목은 *코드/DB 사실에서 검증 가능한 항목*만 기록 — "고객 만족도 확인" 같은 운영 항목은 본 사전 범위 밖.

## 슬롯 채우는 방법

행을 채울 때 다음 형식을 따른다.

```
| 출산지원금 | TCPN887, BIRTH_SUPPORT, SP_CHILD_BIRTH_CD | EHR_HR50/sql/yjungsan/sample/birth_support_2026.sql (사원 T0042~T0045) | sqlplus 접속 후 SELECT 로 사전 확인 → 시나리오 검증 후 ROLLBACK | 2026-04-29 (jinyelimy) |
```

- **위치**: 절대 경로 또는 프로젝트 상대 경로. 운영 DB 안의 테스트 사원이면 `WORK_YY` + `SABUN` 조회 쿼리 명시.
- **사용 방법**: `sqlplus`/`sqlcl` 접속 → SELECT 로 사전 확인 → 검증. **INSERT/UPDATE/DELETE 는 사용자가 직접 수행**(db-read-only 훅이 차단).
- **마지막 검증**: `YYYY-MM-DD (담당자)` — 데이터가 살아 있는지 마지막 확인 시점.

## 신규 테스트 데이터 생성 정책

본 플러그인은 **테스트 데이터를 자동 생성하지 않는다**.

| 행위 | 정책 |
|---|---|
| 테스트 데이터 *위치 조회* | ✅ AI 가 본 사전 + grep 으로 안내 |
| 테스트 데이터 *내용* SELECT 확인 | ✅ AI 가 SELECT 쿼리 작성 (단, 컬럼 검증 게이트 통과 필수) |
| 신규 INSERT/UPDATE/DELETE 스크립트 *작성* | ✅ AI 가 SQL 파일로 작성 가능. 단 **실행은 사용자 수동** |
| 신규 INSERT/UPDATE/DELETE *실행* | ⛔ db-read-only 훅이 자동 차단 |
| 운영 DB 의 테스트 사원 변형 | ⛔ 사용자 명시 승인 + 직접 실행 |

> 이는 연말정산 도메인이 마감·세무 신고와 직결되어 회복 비용이 크기 때문. AI 가 자동으로 더미 INSERT 를 실행하면 오염 발견·롤백이 어렵다.
